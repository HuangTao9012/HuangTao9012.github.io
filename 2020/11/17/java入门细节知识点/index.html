java入门易错点

无标签
发布日期:   2020-11-17

java中的命名规则

  1. 项目名：全部小写 javase
  2. 包名：全部小写 col.sbu.javase.test01
  3. 类名：首字母大写，每个单词首字母大写，选择名词（驼峰原则）TestMethod
  4. 方法名：首字母小写，每个单词首字母大写（驼峰原则） eatApple
  5. 变量名：全部大写；PI
  6. 常量名：全部大写每个单词之间用_隔开 DAN_JIA
  7. 接口名：与类名相同
  8. 关键字都是全部小写

环境变量

  1. 当我们在dos窗口输入命令的时候，window操作系统是如何搜索硬盘上某个指令
    首先会从当前的目录搜索
    如果当前目录没有搜索到结果的话，会从环境变量path中搜索，如果两个都搜索不到，就报错。
  2. path环境变量的作用是什么呢？
    path环境变量其实就是给操作系统指路的。

数据类型

  1. 在java中，数据类型被分为两种，一种的基本数据类型，一种是引用数据类型。
  2. 基本数据类型包括八种，见下表
数据类型 占用空间大小（字节） 取值范围
byte 1 -128~127
short 2 -32768~32767
int 4 -2147483647~2147483648
long 8
float 4
double 8
boolean 1
char 2 0~65535
  1. 一个字节等于8个bit位 1TB=10241024102410248
  2. 关于java中的数字类型，数字都有正负之分，所以在数字的二进制中，有一个二进制位被称为“符号位”。并且这个“符号位”在二进制位的最左边，0表示正数，1表示负数。
    下面以byte为例
    byte类型的最大值：2的7次方减1等于127 最小值是：-128 byte类型可以表示256个不同的数字
    short类型的最大值为：2的15次方减1等于32767 最小值是：-2的15次方 short类型可以表示65536个不同的数字
    int类型的最大值为：2的31次方减1等于2147483647 最小值为-2的32次方

char类型

1、在byte、short、int、long、float、double、boolean、char这八种数据类型中，前七种是很容易表示的，因为他们的底层都是数字，数字和二进制之间存在着一种固定的转换规则。
2、char类型的数据，他表示的是现实社会中的文字，文字和计算机二进制之间默认情况下，是不存在任何转换关系的。
3、为了让计算机可以表示现实社会中的文字相互转换，我们需要人为的干涉。需要人提前制定好“文字”与“二进制”之间的对照关系。这种对照关系，被称为：字符编码
4、计算机最初只支持英文，最先出现的字符编码是：ASCII码
‘a’ –>97
‘A’ –>65
‘0’ –>48
‘a’——-(按照ASCII编码)——->1100001
1100001——-(按照ASCII解码)——->’a’
5、char类型与short类型都是两个字节，但是char类型从0开始没有负数部分。

整型

1、在java语言中，“整数型字面值”被默认当做int类型来处理。如果想让这个字面值被当做long类型来处理的话，需要在后面加上一个L/l;这是java语言规定的。
2、在java语言中整数类型有三种写法：
第一种写法：十进制【默认】10输出10
第二种写法：八进制【需要以0开始】010输出8
第三种写法：二进制【需要以0x开始】0x10输出16
3、最终的输出结果都是十进制

类型转换

1、long x = 123；这行代码的意思是：把一个int类型的数据123转换为long类型。int类型是4个字节，但是long类型是8个字节。这是小容量的数据类型转换为大容量的数据类型是自动类型转换。
2、long x =100L; int y = x;这一行代码编译报错的原因在于，x的值是一个long类型，有8个字节，但是int类型只有4个字节，大容量转换为小的容量需要强制类型转换，需要加上强制类型转换符，但是强制类型转换可能导致精度的缺失。所以强制类型转换需要谨慎使用；
3、强制类型转换的原理（int i= 100；byte a = (byte)i;）
100的原始二进制为：00000000 00000000 00000000 01100100(int类型占用是个字节)
强转之后的二进制为：01100100
强制类型转换的原理是：将左边的二进制位砍掉。所有的强制类型转换都是这样的。
4、分析byte a = 50;
由于在java语言中默认整数的是数据类型是int类型，所以50是一个int类型的数据，int类型占用4个字节，而byte类型是一个字节。将int类型的数据50赋值给byte类型的数据a 是大容量转换为小容量，按照语法规定，需要进行强制类型转换，但是这个地方没有加上强制类型转换的符号。原因在于这个sun公司给byte类型的一种优待，只要是给变量a的值没有超过byte类型的最大值也就是（127）,编译都可以通过，不需要强制类型转换，但是大于127的赋值需要强制类型转换。
5、在计算机中存放的二进制是原始数据的补码形式。正数的源码、反码与补码相同，负数的补码是:将负数取绝对值，求它的二进制，将二进制取反，在加1，得到的结果就是这个负数的补码。
例如：在8位二进制的情况下，求113与-113的源码反码补码
113=96+16+1；
113的源码是：01110001 反码是：01110001 补码是：01110001
-113的源码是：11110001 反码是：10001110 补码是：10001111 移码：00001111
6、分析byte by = (byte)128;
这里128超过了byte类型的最大值，需要强制类型转换，那么计算机底层是怎么做的？
由于计算机存的都是补码的形式，所以需要先知道128在的补码是什么，又因为128是个正数，正数的源码、反码和补码都一样；所以128的源码是：前面有24个0 10000000 反码是：前面有24个0 10000000 补码是：前面有24个0 10000000
类型转换后是：大类型转换为小类型是将左边的部分砍掉 结果为10000000 这个二进制依然是一个补码的形式，这个数值是1开头的，说明他的原始数据是一个负数；负数的补码的计算方法是：负数取绝对值取反在加1；所以想算这个数值的源码是：先减1 10000000-1=01111111 取反 10000000 这个二进制转换为十进制为128 所以这个负数的值为-128
7、分析byte byt = (byte)198;
198转换为二进制为：11000110
11000110-1=11000101
取反：00111010
结果：-(32+16+8+2)=-58
8、8种基本数据类型除开boolean以外其他的都可以相互转化，这8种数据类型的容量的大小顺序是：
Byte<short<int<long<float<double<char
注：任何浮点型的数据不管是占几个字节，都比整型的容量大。
9、多种数据类型进行混合运算，先转换为容量最大的那个类型再进行计算。
